## DNS 记录

在阿里云云解析 DNS 控制台配置：

| 主机记录 | 记录类型 | 记录值 | 说明 |
|---|---|---|---|
| `@` | A | `47.95.8.46` | 裸域 `weid.fun` |
| `*` | A | `47.95.8.46` | 通配符，覆盖 `www.weid.fun` 及其它子域 |

> **坑记**：最初只配了 `*` 通配符，中途把它改成 `@` 的时候，`www.weid.fun` 丢了解析，导致 Let's Encrypt 签发 `www.weid.fun` 的 DCV 返回 NXDOMAIN。后来把 `*` 也补回来才通过。结论：`@` 和 `*` 是两条互不覆盖的记录，裸域不走通配符，需要各自独立配置。

## 常用运维命令

### 查看证书信息

```bash
openssl x509 -in /etc/ssl/weid.fun/fullchain.pem -noout -subject -issuer -dates -ext subjectAltName
```

### 列出 acme.sh 管理的证书

```bash
/root/.acme.sh/acme.sh --list
```

### 手动续签（强制）

```bash
/root/.acme.sh/acme.sh --renew -d weid.fun --force
```

### 重新签发（变更 SAN 等情况）

```bash
/root/.acme.sh/acme.sh --issue --standalone -d weid.fun -d www.weid.fun --force
```

> `--standalone` 模式下 acme.sh 会临时监听 80 端口，执行前确保 80 端口没有其它服务占用（或使用 `--pre-hook` / `--post-hook` 控制 nginx 启停）。

### 重新部署证书到 `/etc/ssl/weid.fun/`

```bash
/root/.acme.sh/acme.sh --install-cert -d weid.fun \
  --key-file       /etc/ssl/weid.fun/privkey.pem \
  --fullchain-file /etc/ssl/weid.fun/fullchain.pem \
  --cert-file      /etc/ssl/weid.fun/cert.pem \
  --ca-file        /etc/ssl/weid.fun/chain.pem
```

当接入 nginx 后，可在 `--install-cert` 追加 `--reloadcmd "systemctl reload nginx"`，acme.sh 每次续签后自动 reload。

## 申请过程踩坑记录

1. **服务器出站网络**：能访问 github.com，但访问 `get.acme.sh` 国内出口 SSL 握手超时。解决：走 GitHub tarball 安装 acme.sh。
2. **DNS 记录 `@` 与 `*` 的区别**：通配符 `*` 不匹配裸域 `@`。申请双 SAN 证书时必须两条记录都存在。
3. **Let's Encrypt 负缓存**：第一次签发时 `www` 记录临时丢失，虽然阿里云权威立即恢复，但 LE 递归 DNS 存在短暂 negative cache（由 SOA minimum TTL 决定，本域名是 600s）。重试一次即通过。
4. **Rate limit 注意**：Let's Encrypt 对失败验证限流 5 次/账号/小时。调试时尽量减少盲目重试。

## 公安备案证书截图获取方式

备案时需要提交 SSL 证书截图。做法：

1. 在服务器上临时启动一个 HTTPS 服务（用 Python 内置 http.server + ssl 即可，无需安装 nginx）
2. 在 Mac 浏览器打开 `https://weid.fun`
3. 点地址栏的锁图标 → 查看证书 → 截图详情页
4. 建议截两张：
   - 浏览器地址栏 + 页面（证明域名在线 + 有锁头）
   - 证书详情窗口（颁发者、SAN、有效期）
5. 截完关掉临时 HTTPS 服务，释放 443

### 临时 HTTPS 服务脚本（保留备用）

```python
# /tmp/https_srv.py
import http.server, ssl

class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        self.send_response(200)
        self.send_header("Content-Type", "text/html; charset=utf-8")
        self.end_headers()
        self.wfile.write(b"<!doctype html><html><head><meta charset=utf-8><title>weid.fun</title></head><body><h1>weid.fun</h1><p>ICP Filed &middot; SSL Enabled</p></body></html>")
    def log_message(self, *a, **k): pass

srv = http.server.HTTPServer(("0.0.0.0", 443), H)
ctx = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
ctx.load_cert_chain("/etc/ssl/weid.fun/fullchain.pem", "/etc/ssl/weid.fun/privkey.pem")
srv.socket = ctx.wrap_socket(srv.socket, server_side=True)
srv.serve_forever()
```

启动：

```bash
nohup python3 /tmp/https_srv.py >/tmp/https_srv.log 2>&1 &
echo $! > /tmp/https_srv.pid
```

关闭：

```bash
kill $(cat /tmp/https_srv.pid)
rm -f /tmp/https_srv.py /tmp/https_srv.pid /tmp/https_srv.log
```
